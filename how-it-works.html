Architecture & Guide

URL Traversal Timer

A high-performance Chrome extension cycling browser tabs through a queue of URLs, running completely offline and in the background.

1 Queue Your Target URLs

The traversal runs inside your **active browser tab**, keeping you inside your current context and session authentication state (e.g. logged-in sessions stay active).

  • Current URL Auto-Add: The active URL is loaded into the queue automatically on launch.
  • Queue Customization: Add custom links or delete existing entries from the list.
  • Reordering Support: Drag and drop URLs to define your custom browsing sequence.

2 Configure Interval Timers

Switch between two advanced timing engines based on your workflow requirements:

  • Fixed Mode: Choose presets (1 min, 3 min, 5 min) or key in a Custom value (in minutes). The system loops exactly on that duration.
  • Random Mode (New!): Specify **Min** and **Max** bounds. The extension dynamically randomizes the wait time between your bounds at the start of each page load. Excellent for organic testing and mimicking human behavior.

3 Control & Monitor

Fire up the traversal and monitor countdown progress directly from Chrome's interface:

  • Circular HUD Display: A circular progress indicator displays remaining seconds in real-time in the popup.
  • Dynamic Badge Counter: Check remaining seconds directly on the extension's toolbar icon badge without opening the popup!
  • Controls: Start, Pause, Resume, and Stop controls give you full manipulation at any point in the cycle.

System Dataflow

The URL Traversal Timer employs a decoupled frontend and background worker architecture, using **Chrome Alarms** for persistent thread scheduling and **Plasmo Local Storage** as a reactive single source of truth.

📱

Popup Client (popup.tsx)

Standard React components with Tailwind styling. Dispatches start, pause, and stop actions to the background worker via Plasmo Messaging pipelines, and hooks into stored states for real-time reactive renders.

↓
⚡

Background Message Receiver (messages/startTraversal.ts)

Receives asynchronous runtime payload from popup threads (URLs, intervals, random toggles, and boundary limits) and relays commands directly to the core traversal engine.

↓
⚙️

State & Timing Controller (traversalEngine.ts)

Core state orchestrator. Persists active status details into Chrome local storage. Computes exact next run timestamps (with randomized calculations on the fly when Random mode is enabled) and schedules the low-level alarms.

↓
🔔

Persistent Scheduler (chrome.alarms & chrome.tabs)

Chrome Alarm triggers at the precise scheduled milliseconds. The service worker wakes up, navigates the active tab to the next URL, computes the next interval duration, updates Chrome's extension badge UI, and schedules the next cycle seamlessly.

Under The Hood Note: Standard JavaScript `setTimeout` and `setInterval` execution contexts are aggressively throttled or completely killed when Chrome extension popups close or background pages suspend. By offloading scheduling to chrome.alarms, the timer maintains 100% reliable execution and millisecond accuracy across tab states without keeping heavy background pages alive.

Local Storage Schema

This extension runs entirely locally on your machine and uses zero backend dependencies. The data layout inside `chrome.storage.local` is structured as follows:

{
  // Current active traversal state (Key: "utt:tab:[tabId]:state")
  "utt:tab:1320491:state": {
    "urls": [
      "https://google.com",
      "https://github.com"
    ],
    "currentIndex": 1,
    "isRunning": true,
    "isPaused": false,
    "nextRunAt": 1779838029000,
    "intervalMs": 85000,       // Computed randomly for the current cycle
    "isRandom": true,          // Random traversal enabled
    "minIntervalMs": 60000,    // 1-minute floor
    "maxIntervalMs": 180000    // 3-minute ceiling
  },

  // Saved URL queue per page (Key: "utt:page:[encodedUrl]:urls")
  "utt:page:https%3A%2F%2Fgoogle.com:urls": [
    "https://google.com",
    "https://github.com"
  ]
}
